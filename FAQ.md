# FAQ

**Q: Are you claiming to have deciphered Linear A?**
No. This project makes no translation claims whatsoever. We measure
structure: which regularities in the corpus are real (survive permutation
null models) and which are statistical mirages. The output is candidates
and constraints, not readings.

**Q: What is the method, in ten lines?**
1. Freeze a validated corpus (1721 inscriptions) — never edit it.
2. For every hypothesis, define the statistic *before* looking.
3. Build an explicit null model: shuffle labels, permute signs within
   positional classes, resample subsets (seed=42, R=10 000).
4. Report the observed value, the null distribution, and an exploratory
   p-value; no multiple-comparison shopping.
5. Prefer paired and stratified designs (same stem, same length class).
6. Replicate across independent archives before trusting a marker.
7. Rerun key tests on "reliable readings only" (epigraphic-doubt layer).
8. Publish negative results with the same prominence.
9. Every number in the report is regenerated by a seeded script.
10. A clean-clone stress test verifies byte-level reproducibility.

**Q: What are the strongest results so far?**
(a) Internal confirmation that Linear B sound values transfer to Linear A
(vowel-sign behaviour, consonant-sharing alternations); (b) a
statistically controlled onomastic layer shared with Greek Linear B
archives — three Cretan toponyms and six personal-name candidates confirmed
in name slots of Knossos records; (c) a calibrated morphological inventory
(-JA, A-, PI-) with a register function for -JA; (d) a site-specific
"geography slot" in the libation formula; (e) commodity-dependent
metrology (binary grain grid vs duodecimal oil grid).

**Q: What did NOT work?**
Short-word Linear A ≡ Linear B "identical word" lists are chance-level
(this includes most of the popular lists circulating online). Single
interspersed signs (the HT 118 "KI column") are not a system. Scribal
idiolect does not explain spelling variants. A- prefix has no measurable
document function yet. Substitution tournaments for unknown signs are
non-resolving at the current size of the comparison corpus. We document
each of these negatives in the report.

**Q: Why should anyone trust a hobby project?**
Because nothing rests on trust: the corpus is frozen and validated against
a hand-checked sample, every script is seeded, the full battery reruns
from scratch with an empty `git diff`, and the report separates
"significant", "suggestive" and "retracted" explicitly.

**Q: What data are you waiting for?**
(1) The full edition of the Knossos ivory sceptre (Anetaki plot; the
longest Linear A text, ~119 signs) — our formula slot engine is validated
and ready. (2) A machine-readable full Linear B corpus with record
contexts (e.g. LiBER) — it would roughly double the name-slot comparison
set and could make the unknown-sign substitution tournament resolving.
(3) A digitized Cypro-Minoan corpus for the continuity test.

**Q: Can I reuse the code/data?**
Code is MIT-licensed. Derived data tables carry the credits of the
underlying editions (see DATA_SOURCES.md); raw third-party files are not
redistributed — a fetch script pulls them from the pinned upstream source.

**Q: How can I help?**
Point us to machine-readable editions (Linear B, Cypro-Minoan, new Linear A
finds), check our statistics, or open an issue with a falsifiable idea —
the null-model harness makes testing cheap.
