# Этрусский язык: переносимость нашего метода и честный прогноз

*Меморандум по запросу владельца проекта; пилотные числа —
etruscan_pilot.py / etruscan_pilot.log; данные — Larth-Etruscan-NLP
(Vico & Niculae 2023) + CIEP-конкорданс.*

## 1. Чем этрусская задача отличается от линейного А

| | Линейное A | Этрусский |
|---|---|---|
| Чтение знаков | было под вопросом (мы подтвердили перенос из LB) | НЕ проблема: греческий алфавит, читается надёжно |
| Понимание | ~0 слов доказано | ~200–400 слов понято (контексты, билингва из Пирги, глоссы) |
| Корпус | 1721 надпись, ~7.5 тыс. знаков | ~13 000 надписей; в нашем датасете 5842 с текстом |
| Длинные тексты | нет (макс. ~120 знаков — скипетр) | есть: Liber Linteus (~1330 слов!), Tabula Capuana, Cippus Perusinus, Tabula Cortonensis |
| Валидационный набор | НЕТ (нечем мерить «понимание») | ЕСТЬ: 2445 надписей с переводами в датасете |
| Родство языка | неизвестно | изолят (тирсенская семья: +лемносский, ретийский) |

Ключевое: в LA мы решали задачу «есть ли структура и переносимы ли чтения»;
в этрусском задача сразу следующего уровня — «расширить ПОНИМАНИЕ», и для
неё существует то, чего в LA нет в принципе: **обучающая/проверочная
выборка** (слова с надёжными значениями). Это превращает «дешифровку
смысла» из философской проблемы в измеримую задачу с метрикой качества.

## 2. Пилот: метод переносится один-к-одному (уже проверено)

Прогнали наши инструменты на 5842 надписях (etruscan_pilot.log):
- **Реестр операторов** работает сразу: mi «я» — 89% начальной позиции
  (формулы «говорящих предметов»), lupu «умер» — 80% финальной,
  zilath «магистрат» — начальная, clan «сын» — никогда не начален.
  Профили — как у наших KU-RO/SI-RU-TE, только со ЗНАЧЕНИЯМИ.
- **Формульность** экстремальна: топ-10 шаблонов покрывают 96% надписей
  (короткие эпитафии/посвящения) — слот-машина §AX применима напрямую.
- **Парадигмы** массивны: генитив -s — 876 типов, -al — 317, -sa — 214
  (у нас в LA лучший маркер имел 9 основ) — калибровка маркеров §L будет
  работать с огромным запасом мощности; паттерн «X-al + clan» («сын
  такой-то») ловится автоматически (×11 в сыром пилоте).
- Гапаксы — 78% (сопоставимо с LA): проблема одноразовых слов остаётся,
  но она смягчена переводимым ядром и длинными текстами.

## 3. Можно ли «понять» этрусский статистикой? Честный ответ

**Полностью — нет.** Ограничения фундаментальны: язык-изолят (внешних
опор нет), 78% слов — гапаксы (их значения не восстановимы из
распределения), а большая часть корпуса — имена собственные с формулами
родства. Никакая статистика не извлечёт смысл слова, встреченного один
раз без контекста.

**Существенно продвинуться — да, и вот где именно:**
1. **Морфология до конца.** Падежная система, цепочки генитивов,
   глагольные -ce/-χe: на 5000+ типов наши калиброванные методы (поддержка
   парадигм, семейный контроль, репликация по городам/векам — у датасета
   есть датировки!) дадут полную формальную грамматику имён и, вероятно,
   глагола. Реалистично: карта морфем с доказательной базой, которой у
   этрускологии в количественной форме нет.
2. **Дистрибутивная семантика с учителем.** Главная новация, невозможная
   в LA: обучить распределительные представления (контексты, слоты,
   совместная встречаемость с известными словами) на ~300 словах с
   надёжными значениями, проверить на отложенной части (метрика!), и
   применить к неизвестным. Цель — не «перевод», а РАНЖИРОВАННЫЕ гипотезы
   значений с калиброванной точностью («слово X с вероятностью ~0.7 из
   семантического класса „сосуд/дар“»). Реалистичный выход: +100–300 слов
   с вероятностными значениями к нынешним ~300 понятых.
3. **Liber Linteus как суперформула.** Ритуальный календарь с высокой
   повторяемостью — идеальная цель слот-машины: разложить на слоты
   (даты / божества / действия / приношения), связать слоты с известными
   словами (месяцы, боги известны частично) — связное СТРУКТУРНОЕ чтение
   длиннейшего текста (не пословный перевод, но «что где происходит»).
   Требуется оцифровка транслитерации LL (в открытом виде не найдена;
   есть научные издания — вопрос набора/запроса).
4. **Числа, меры, календарь** — наш метрологический аппарат применим
   напрямую (avils-числа возрастов уже размечаются).

**Оценка вероятностей (субъективно):** полная дешифровка «читать любой
текст» — <5% (упирается в гапаксы и изолят); «широкое понимание»
(грамматика полностью + 500–700 слов + структурное чтение больших
текстов) — **40–60% при систематической работе**, потому что все
инструменты уже есть, данные машиночитаемы, и — решающее отличие —
существует метрика успеха. Этрусский для нашего метода — задача
*благодарнее* линейного A: там мы строили фундамент вслепую, здесь
фундамент (чтение + опорный словарь) уже стоит.

## 4. План пилота-продолжения (если решим заходить)

1. Заморозить этрусский корпус по нашему протоколу (Larth+CIEP+ETP,
   валидация, supplement-механизм) — 1 этап.
2. Реестр операторов v1 + слот-шаблоны эпитафий/посвящений с
   LOO-валидацией на переводах — 1 этап.
3. Калибровка морфем (суффиксы/энклитики) с семейным контролем и
   репликацией по городам и векам — 1–2 этапа.
4. Дистрибутивная семантика с учителем: train/test на переведённых,
   отчёт с калибровкой качества — 2–3 этапа.
5. Оцифровка Liber Linteus (набор из издания или запрос) → слот-разбор.
